# CLAUDE.md

## Project Overview

Video Manager -- full-stack app for managing media from Immich/OneDrive. Tag assets, manage people (face recognition), create collections, export to Shotcut MLT. Registration is invite-only via admin invite links.

## Tech Stack

- **Backend**: .NET 10, ASP.NET Core, C# 13, EF Core 10, PostgreSQL
- **Frontend**: Next.js 16, React 19, TypeScript 5.9, Tailwind CSS 4, shadcn/ui
- **State**: Zustand (client) + React Query (server)
- **Orchestration**: .NET Aspire 13.1.0 (uses podman)
- **Auth**: JWT with BCrypt, role-based (User/Admin)

## Running the Application

**Always create a plan** before implementing and save it in the `plans/` folder.
**Always implement features** in small increments and get verification from the user.

**Use Aspire** for local development (starts API, frontend, PostgreSQL, pgAdmin, dashboard):
```bash
cd VideoManager/VideoManager.AppHost && dotnet run
```

**Frontend standalone**: `cd video-manager-frontend && npm run dev`

## Common Commands

```bash
# Backend: EF Core migrations (from VManBackend directory)
dotnet ef migrations add {Name}

# Frontend
npm run dev                  # Dev server
npm run dev:with-gen         # Generate API client then dev
npm run lint                 # ESLint
npm run generate:client      # Regenerate Kiota API client
npm run test:e2e             # Playwright E2E tests
npm run test:e2e:ui          # Interactive test mode
```

## Critical Rules

- **Never** run `npm run build` to verify -- changes are hot reloaded
- **Never** edit `video-manager-frontend/src/lib/api/client.ts` manually (generated by Kiota)
- Don't use FluentValidation or MediatR -- custom implementations exist in `Mediator/`
- Don't use `.WithOpenApi()` (deprecated) -- use `.WithTags()`
- Don't use attribute routing (`[Route]`, `[Authorize]`) -- use minimal APIs
- Don't store server data in Zustand -- use React Query for all API data
- Use `import { useToast } from '@/hooks/use-toast';` for toast notifications

## Project Structure

```

├── VideoManager/
│   ├── VideoManager.AppHost/          # Aspire orchestration (start here)
│   ├── VideoManager.ServiceDefaults/  # Shared Aspire defaults
│   └── VManBackend/                   # ASP.NET Core API
│       ├── Common/Data|Models/        # DbContext + entity models
│       ├── Features/                  # Vertical slices (Admin, Auth, Collections, Items, People, Sync, Tags)
│       ├── Endpoints/                 # Minimal API route mappings
│       ├── Infrastructure/            # JWT, Immich client, providers, sync engine
│       ├── Mediator/                  # Custom CQRS (not MediatR)
│       └── Migrations/
├── video-manager-frontend/            # Next.js 16 frontend
│   ├── src/app/                       # App Router ((auth), (dashboard), complete-profile)
│   ├── src/components/                # React + shadcn/ui components
│   ├── src/lib/                       # API client, hooks, stores, validations
│   └── tests/                         # Playwright E2E (Page Object Model)
├── plans/                             # Implementation plans (create before implementing)
└── .claude/rules/                     # Scoped rules (auto-loaded per file path)
```

Detailed patterns, API reference, code templates, and conventions are in `.claude/rules/` -- they load automatically when editing relevant files.
